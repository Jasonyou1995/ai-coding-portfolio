## 🎯 阶段规划

### Phase 1: 基础工具 (第 1-4 周)

**目标**: 掌握 Python 核心编程，能独立完成 CLI 工具和数据处理脚本。

| 周次 | 项目 | 核心技能 | 产出物 |
|:---:|:---|:---|:---|
| 1 | Smart Mailbox | FastAPI 基础、REST API 设计 | 可运行的邮箱 API |
| 2 | Code Reviewer AI | Git Hook、文件处理、OpenAI API | Git 自动代码审查 |
| 3 | Config Manager | YAML/JSON 解析、Click、多环境 | 配置管理 CLI |
| 4 | Log Analyzer | 正则表达式、Pandas、数据可视化 | 日志分析工具 |

**学习检查点**:
- [ ] 熟练使用 Python 类型提示
- [ ] 理解异步编程基础 (async/await)
- [ ] 掌握 pytest 测试框架
- [ ] 熟悉 Git 工作流

---

### Phase 2: 网络服务 (第 5-12 周)

**目标**: 能设计和实现高并发 Web 服务，理解网络编程核心概念。

| 周次 | 项目 | 核心技能 | 产出物 |
|:---:|:---|:---|:---|
| 5-6 | URL Shortener | 哈希算法、Bloom Filter、缓存 | 高并发短链服务 |
| 7-8 | Chat Server | WebSocket、实时通信、消息广播 | 在线聊天室 |
| 9-10 | API Gateway | 反向代理、限流、认证、路由 | API 网关 |
| 11-12 | File Service | 分片上传、对象存储、断点续传 | 文件存储服务 |

**学习检查点**:
- [ ] 理解 HTTP 协议细节
- [ ] 掌握数据库设计（索引、事务）
- [ ] 理解缓存策略和 Redis
- [ ] 能进行基础性能优化

---

### Phase 3: AI 应用 (第 13-22 周)

**目标**: 深入理解 LLM 应用开发，能构建 RAG 系统和 Agent。

| 周次 | 项目 | 核心技能 | 产出物 |
|:---:|:---|:---|:---|
| 13-15 | RAG System | 向量数据库、Embedding、检索 | 智能问答系统 |
| 16-18 | Agent Framework | ReAct、Tool Use、Planning | 自主任务 Agent |
| 19-20 | MCP Server | MCP 协议、Context 管理 | MCP 兼容服务 |
| 21-22 | AI Translator | Prompt Engineering、流式输出 | 智能翻译 API |

**学习检查点**:
- [ ] 理解 Embedding 和向量检索
- [ ] 掌握 Prompt Engineering
- [ ] 理解 Agent 架构模式
- [ ] 能进行 Token 优化

---

### Phase 4: 系统架构 (第 23-30 周)

**目标**: 理解分布式系统原理，能设计和部署微服务架构。

| 周次 | 项目 | 核心技能 | 产出物 |
|:---:|:---|:---|:---|
| 23-24 | Distributed Lock | Redis Redlock、一致性 | 分布式锁实现 |
| 25-26 | Message Queue | Kafka、消息可靠性、顺序性 | 消息队列系统 |
| 27-28 | Rate Limiter | 滑动窗口、令牌桶、集群限流 | 分布式限流器 |
| 29-30 | K8s Deployment | 容器化、K8s、Helm、CI/CD | 完整 K8s 部署 |

**学习检查点**:
- [ ] 理解 CAP 理论
- [ ] 掌握分布式一致性
- [ ] 能设计高可用架构
- [ ] 熟悉云原生技术栈

  └── 下周预习
```

## 💡 学习建议

### 1. 费曼学习法

每个项目完成后，尝试用自己的话解释：
- 这个项目的核心挑战是什么？
- 我是如何解决的？
- 如果重新做，我会怎么改进？

### 2. 刻意练习

- 每个项目至少迭代 3 个版本
- 保留"不完美"的代码和修复记录
- 主动寻找代码审查机会

### 3. 记录习惯

- ...
```

## 🎓 完成标准

每个项目必须满足以下标准才算"完成":

- [ ] 代码可运行，有 README 说明
- [ ] 包含单元测试，覆盖率 > 80%
- [ ] 有技术决策文档 (ADR)
- [ ] 有迭代记录（至少 2 轮）
- [ ] 有性能测试数据
- [ ] 通过隐私检查（无敏感信息）
