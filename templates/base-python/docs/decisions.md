# 技术决策记录 (Architecture Decision Records)

## ADR-001: 为什么选择 FastAPI?

**状态**: 已接受  
**日期**: 2024-XX-XX

### 背景

需要选择一个 Web 框架来构建 API 服务。

### 考虑选项

| 选项 | 优点 | 缺点 |
|:---|:---|:---|
| Flask | 简单、生态成熟 | 异步支持弱，需额外配置 |
| Django | 功能齐全 | 太重，不适合小型服务 |
| FastAPI | 原生异步、自动生成文档、类型提示 | 相对较新 |

### 决定

选择 **FastAPI**。

理由：
- 原生支持异步，性能更好
- 自动 OpenAPI 文档生成
- 基于 Pydantic 的类型验证
- 学习曲线平缓

### 后果

- 需要团队成员学习 Python 类型提示
- 需要关注异步编程模式

## ADR-002: 项目结构

**状态**: 已接受  
**日期**: 2024-XX-XX

### 决定

采用 `src/` 布局而非扁平布局。

### 理由

- 更清晰的项目结构
- 避免导入路径问题
- 更好的 IDE 支持
